# 탐색적 데이터 분석 (Exploratory Data Analysis)

## 1. 탐색적 데이터 분석이란?

### 고전적인 통계학 vs 탐색적 데이터 분석(EDA)

* **고전적인 통계학**: 적은 표본(Samples)을 가지고 더 큰 모집단에 대한 결론을 도출하기 위한 일련의 복잡한 과정.
* **탐색적 데이터 분석 (EDA)**:
* 데이터의 본질을 이해하고, 데이터에 숨겨진 패턴을 찾기 위해 데이터를 시각화하고 요약하는 과정.
* 데이터 시각화 도구(Matplotlib, Seaborn 등) 활용.
* 가정에 얽매이지 않음 (고전적 통계학에서는 주로 가정을 세움).
* 이상치 탐지 (특이한 값 확인).
* 기술 통계 수행 (고전적 통계와 비슷).
* *존 튜키(John Tukey)*가 제안함.



### EDA 과정에서 파악할 요소

* 데이터 사이즈는 어느 정도인지
* 학습 데이터와 테스트 데이터는 어떻게 분리가 되어있는지
* 결측값은 어느 정도인지
* 라벨이 있는 데이터라면 분포는 어떻게 되어있는지
* 데이터의 특이점이 있는지

### 분석 사이클

1. 데이터에 대한 가설 혹은 의문 설정
2. 시각화, 통계량 혹은 모델링을 통한 가설 검정
3. 결론을 통해 다시 새로운 가설 혹은 문제 도출
* *이 과정을 반복하며 기본적인 데이터를 이해하는 것이 목적*

## 2. 정형 데이터와 테이블 데이터의 이해

### 정형 데이터 요소 (Data Types)

데이터를 분석하고 예측을 모델링할 때, 시각화/해석/모델 결정 등에 데이터의 종류가 중요한 역할을 하므로 분류가 필요함.

1. **수치형 데이터 (Numerical)**
* **연속형 데이터**: 무한한 값을 가질 수 있는 수치 데이터 (예: 풍속, 지속시간 등)
* **이산형 데이터**: 특정한 정수값을 가질 수 있는 수치 데이터 (예: 사건의 발생 빈도 등)


2. **범주형 데이터 (Categorical)**
* **명목형 데이터**: 순서의 의미가 없는 범주 데이터 (예: TV 스크린 종류, 도시명 등)
* **순위형 데이터**: 순서의 의미가 있는 범주 데이터 (예: 평점 1~5, 선호도 등)



### 테이블 데이터 (Table Data)

* 데이터 분석에서 가장 대표적으로 사용하는 데이터 형태.
* **DataFrame**: 통계와 머신러닝 모델에서 가장 기본이 되는 테이블 형태의 데이터 구조.
* **Feature**: 일반적으로 테이블의 각 열(Column)을 의미.
* **Record**: 일반적으로 테이블에서 하나의 행(Row)을 의미.

## 3. 정형 데이터 EDA 과정 살펴보기 (타이타닉 데이터 예시)

타이타닉 데이터를 사용하여 EDA를 진행하는 예시입니다.

### 1) 데이터 확인 및 준비

* 타이타닉 승객 데이터에서 생존(Survived)에 영향을 주는 요인을 탐색.
* 생존자 비율, 특성과 생존의 연관성(Pclass, Sex, Age, Fare, Family 등), 결측/왜곡 확인.
* **프로세스**: 질문 -> 요약(표) -> 시각화 -> 인사이트의 반복.

**[코드] 라이브러리 임포트 및 데이터 로드**

```python
import numpy as np
import pandas as pd
import matplotlib.pyplot as plt
import seaborn as sns

plt.rc('font', family='Malgun Gothic') # 한글 폰트 설정

df = pd.read_csv("titanic.csv")

```

**데이터 컬럼 정보**

* **PassengerId**: 승객 ID (단순 식별자, 예측에 비중 없음)
* **Survived**: 생존 여부 (0: 사망, 1: 생존) - *Target Variable*
* **Pclass**: 티켓 클래스 (1: 1등석, 2: 2등석, 3: 3등석)
* **Name**: 이름
* **Sex**: 성별 (male, female)
* **Age**: 나이
* **SibSp**: 함께 탑승한 형제/자매, 배우자의 수
* **Parch**: 함께 탑승한 부모, 자녀의 수
* **Ticket**: 티켓 번호
* **Fare**: 운임 요금
* **Cabin**: 객실 번호
* **Embarked**: 탑승 항구 (C: 쉘부르, Q: 퀸즈타운, S: 사우샘프턴)

**[코드] 데이터 정보 확인**

```python
df.info()

```

* **분석 결과**:
* 총 891개의 엔트리.
* `Age`, `Cabin`, `Embarked`에 결측치(Null)가 존재함.
* 특히 `Cabin`의 결측치가 상당히 많아 그대로 사용하기 어려움.

**수치형 변수 분포 확인**

* **Age**: 20~40대 구간에 승객이 많이 분포. 어린 나이와 고령 구간은 적음. 결측치 전처리 필요.
* **Fare**: 오른쪽으로 치우친 비대칭 분포. 소수의 고액 요금 승객 존재.

**이산형 변수 분포 확인 (SibSp, Parch)**

* 대부분의 승객이 `SibSp`, `Parch` 값이 0 또는 1에 집중되어 있음 (혼자 탑승한 경우가 많음).

### 2) Feature Exploration (특성 탐색)

**Survived (타겟 변수)**

* **코드**: `df['Survived'].value_counts()`
* **분석**:
* 0(사망): 549명 (61.6%), 1(생존): 342명 (38.4%)
* 사망자의 비율이 더 높음. 균형 잡힌 데이터가 아니므로 유의 필요.



**Pclass (티켓 등급)**

* **코드**: `df['Pclass'].value_counts()`
* **분석**:
* 3등석 승객이 가장 많음.
* **생존율**: 1등석의 생존율이 가장 높고, 등급이 낮아질수록 생존율 감소.
* 3등석에서 사망자 수가 가장 많이 발생. 생존 여부와 강한 관련성이 있는 변수.



**Sex (성별)**

* **분석**:
* 남성 탑승객 비율이 여성보다 높음 (약 65:35).
* **생존율**: 사망자 중 남성 비율이 매우 높음. 반면 여성의 생존율이 높음.
* 위급 상황에서 여성을 우선 대피시켰을 것으로 추측됨. 매우 중요한 Feature.



**Age (나이)**

* **분석**:
* 탑승객 나이 분포는 20~40대가 가장 많음.
* 연령대별로 생존 비율이 달라지는지 추가 확인 필요.



**SibSp & Parch (동반자)**

* **SibSp**: 형제자매/배우자와 함께 탑승하지 않은 승객(0)이 가장 많음.
* **Parch**: 부모/자녀와 함께 탑승하지 않은 승객(0)이 가장 많음.
* 동반 인원 수에 따라 생존 여부에 차이가 있는지 추가 분석 필요.

**Embarked (탑승 항구)**

* **분석**:
* S항구, C항구, Q항구 3종류. S항구 탑승객이 압도적으로 많음.
* S항구 탑승객의 사망률이 높아 보임. 이는 S항구 탑승객 중 3등석 비중이 높은 것과 관련 있을 가능성 시사.



**Fare (요금)**

* **분석**:
* 높은 요금일수록 생존 확률이 높아지는 경향.
* 이는 높은 등급(Pclass) 및 사회적 위치를 반영한 결과일 수 있음.



**Ticket (티켓 번호)**

* **코드**: `df['Ticket'].unique()`
* **분석**:
* 티켓 번호에 대한 특별한 패턴을 찾기 어려움 (High Cardinality).
