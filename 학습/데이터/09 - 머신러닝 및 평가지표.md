# 머신러닝 개요 및 평가지표

## 1. 인공지능의 기본 개념

### AI, 머신러닝, 딥러닝의 관계

* **인공지능(AI)**: 인간의 지능을 인공적으로 만든 것. (가장 포괄적인 개념)
* 1단계: 규칙 기반의 알고리즘


* **머신러닝(ML)**: 데이터를 기반으로 학습하는 것.
* 2단계: 데이터 기반의 학습 (결정 트리, 선형회귀, 퍼셉트론, SVM 등)


* **딥러닝(DL)**: 심층 신경망(DNN)을 사용하는 머신러닝의 하위 분야.
* 3단계: CNN, RNN, GAN, 트랜스포머 등



### 머신러닝의 정의

* **Arthur Samuel (1959)**: 컴퓨터가 명시적으로 프로그래밍되지 않아도 스스로 학습할 수 있는 능력을 부여하는 학문 분야.
* **Tom Mitchell (1998)**: 컴퓨터 프로그램이 어떤 작업 와 성능 척도 에 대해 경험 로부터 학습한다고 할 때, 경험 를 통해  작업에 대한 성능이 로 측정했을 때 향상되는 경우.
* **경험(E)**: 데이터, 과거의 사례
* **작업(T)**: 해결하려는 문제, 과제
* **성능 척도(P)**: 얼마나 잘했는지 평가하는 기준

## 2. Feature Engineering (피처 엔지니어링) 주요 기법

데이터를 모델이 잘 학습할 수 있도록 가공하는 과정입니다.

### 1) 피처 선택 (Feature Selection)

모델 성능에 중요한 변수만 선택하고 불필요한 변수는 제거합니다.

* **상관계수 분석**: 상관관계가 높은 변수 제거
* **분산 기반 선택**: 변화가 거의 없는 변수 제거
* **RFE (Recursive Feature Elimination)**: 모델 성능에 가장 중요한 변수 선택

### 2) 피처 생성 (Feature Creation)

기존 데이터를 활용하여 새로운 변수를 생성합니다.

* **예시**:
* 날짜 데이터  "요일", "주말 여부"
* 판매 데이터  "가격  판매량" (총 매출)



### 3) 범주형 데이터 인코딩 (Categorical Encoding)

문자 데이터를 숫자로 변환하여 모델이 이해할 수 있도록 합니다.

* **원-핫 인코딩 (One-Hot Encoding)**: 카테고리를 0과 1로 변환
* **라벨 인코딩 (Label Encoding)**: 카테고리를 숫자로 변환 (예: "Red"  1, "Blue"  2)

### 4) 결측값 처리 (Handling Missing Values)

* **삭제**: 결측치가 적으면 해당 행 제거
* **대체**: 평균, 중앙값, 최빈값으로 채우기
* **예측 모델 사용**: 머신러닝을 활용하여 결측값 예측

## 3. 머신러닝의 분류

### 1) 지도학습 (Supervised Learning)

정답(Label)이 있는 데이터를 학습합니다.

* **회귀 (Regression)**: 연속적인 값을 예측 (예: 집 가격 예측)
* Data: Size, Lot size  Price


* **분류 (Classification)**: 범주를 예측 (예: 이미지 분류 - 고양이, 강아지, 비행기 등)
* 객체 위치 지정 및 탐지 (Bounding boxes)



### 2) 비지도학습 (Unsupervised Learning)

레이블(정답)이 없는 데이터에서 흥미로운 구조를 발견합니다.

* **군집화 (Clustering)**: 유사한 데이터끼리 그룹화 (예: 유전자 클러스터링)

### 3) 자기지도학습 (Self-Supervised Learning)

* 적은 양의 데이터셋을 통해 학습된 기계 학습 모델 (예: Large Language Model)
* 다양한 목적으로 활용 가능 (예: ChatGPT)

### 4) 강화학습 (Reinforcement Learning)

시행착오를 통해 보상을 최대화하는 방향으로 학습합니다.

* **구성 요소**: 에이전트(Agent), 환경(Environment), 행동(Action), 보상(Reward)
* **과정**: 행동  환경 변화  보상  학습 (피드백을 통한 개선)
* 예: 알파고, 게임 플레이

## 4. 모델 성능 평가 (Evaluation)

실제값과 모델에 의해 예측된 값을 비교하여 오차(차이)를 구하는 것입니다.

### 1) 회귀 모델 평가 방법

* **MAE (Mean Absolute Error)**: 실제 정답 값과 예측 값의 차이를 절대값으로 변환하여 평균.
* **MSE (Mean Squared Error)**: 실제 정답 값과 예측 값의 차이를 제곱한 뒤 평균을 구함.
* **수식**:


* 이상치(Outlier)에 매우 민감함 (큰 오차에 큰 페널티).
* 값이 낮을수록 좋은 모델.


* **R2 Score (결정 계수)**: 데이터의 변동성 중에서 모델이 설명할 수 있는 비율. 1에 가까울수록 좋음.
* 
 (SSR: Sum of Squares Regression, SST: Total Sum of Squares)



### 2) 분류 모델 평가 방법

* **오차행렬 (Confusion Matrix)**
* **TP (True Positive)**: 참을 참으로 예측 (정답)
* **TN (True Negative)**: 거짓을 거짓으로 예측 (정답)
* **FP (False Positive)**: 거짓을 참으로 예측 (오답, 1종 오류)
* **FN (False Negative)**: 참을 거짓으로 예측 (오답, 2종 오류)


* **정밀도 (Precision)**: 모델이 "참"이라고 예측한 것 중 실제로 참인 비율.
* 
* 예: 스팸 메일 분류 (일반 메일을 스팸으로 분류하면 안 됨)


* **재현율 (Recall)**: 실제 "참"인 것 중에서 모델이 참으로 예측한 비율.
* 
* 예: 암 환자 진단 (암 환자를 놓치면 안 됨)


* **Precision과 Recall의 Trade-off**
* 임계값(Threshold)을 조절하면 두 지표는 반비례 관계를 가짐.


* **F1-Score**
* 정밀도와 재현율의 조화 평균. 데이터가 불균형할 때 유용함.
* 


* **ROC 곡선과 AUC**
* TPR(True Positive Rate, 재현율)과 FPR(False Positive Rate)의 관계 그래프.
* 곡선이 좌상단에 붙을수록(AUC 면적이 넓을수록) 좋은 모델.

## 5. 벡터의 정의와 개념

### 벡터의 정의

* 선형대수학의 기본 요소로, **크기와 방향**을 가짐.
* 좌표 평면에서 점을 나타낼 때 $(x, y)$와 같은 형태로 표현.
* 머신러닝과 데이터 분석 등에서 데이터를 표현하는 기본 단위로 사용.
* 파이썬에서는 숫자로 이루어진 배열(리스트) 형태로 표현.

### 데이터를 표현하는 방법 (차원별)

* **스칼라 (Scalar)**: 하나의 숫자 (0차원)
* **벡터 (Vector)**: 여러 개의 숫자가 나열된 1차원 배열
```text
[ 1 ]
[ 2 ]

```


* **행렬 (Matrix)**: 숫자의 행과 열로 구성된 2차원 배열
```text
[ 1  2 ]
[ 3  4 ]

```


* **텐서 (Tensor)**: 3차원 이상을 포함하는 행렬보다 높은 다차원 배열
```text
[ [ 1  2 ]   [ 5  6 ] ]
[ [ 3  4 ]   [ 7  8 ] ]

```



### 텐서의 활용

* 딥러닝, 머신러닝에 적용하여 데이터를 표현함.
* 이미지 데이터(RGB 채널 등)를 처리할 때 다차원 배열(텐서) 사용.
* **Code Example (영상 내 표기)**:
```text
First Tensor : tensor([1., 2., 3.])
Second Tensor : tensor([4., 5., 6.])
Third Tensor : tensor([7., 8., 9.])

Join tensors in the 0 dimension
tensor([[1., 2., 3.],
        [4., 5., 6.],
        [7., 8., 9.]])

```
